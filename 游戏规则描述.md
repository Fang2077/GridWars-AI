## 一、游戏概述

这是一个双 AI \单人+AI\双人回合制策略游戏。

* 地图大小：10 × 10 网格
* 阵营：红队（正数阵营）、黑队（负数阵营）
* 单位类型：营地（Base）、战士（Warrior）
* 核心机制：所有单位以“带符号整数点数”作为唯一属性
* 阵营由点数符号决定

---

## 二、基本定义

### 1. 阵营规则

* 点数 > 0 → 红队
* 点数 < 0 → 黑队
* 点数 = 0 → 单位立即消失

---

### 2. 单位类型

#### （1）营地（Base）

功能：

* 生产战士
* 可被攻击
* 可升级
* 用于胜负判定

#### （2）战士（Warrior）

功能：

* 可移动
* 可与其他单位发生冲突
* 可攻击敌方营地
* 可升级己方营地

---

## 三、初始状态

* 地图为空棋盘（可能存在障碍，若无特别说明则默认无障碍）
* 红队在左上角 (0,0) 拥有一个营地
* 黑队在右下角 (9,9) 拥有一个营地
* 初始营地点数默认为：

  * 红队：+1
  * 黑队：-1

---

## 四、回合流程

每回合按以下顺序执行：

### Step 1：掷骰

* 当前行动方获得一个 1–6 的随机整数 D

---

### Step 2：行动选择（必须执行且只能选择一种）

AI 可选择以下行为之一：

---

#### A. 建造战士

条件：

* 只能在己方营地的上下左右四个相邻格
* 该位置必须为空且非障碍

效果：

* 生成一个点数 = D 的战士

  * 红队生成 +D
  * 黑队生成 -D

---

#### B. 移动战士

* 选择一个己方战士
* 向上下左右移动一格
* 目标格必须合法（地图内）

移动后立即结算冲突（见冲突规则）

---

#### C. 建造营地（仅当 D = 6 时可选）

条件：

* 当前阵营场上至少存在一个战士
* 可在任意一个己方战士的上下左右四格建造
* 目标格必须为空

效果：

* 红队生成 +1 营地
* 黑队生成 -1 营地

---

## 五、冲突规则（核心规则）

### 1. 战士 vs 战士

当两个战士进入同一格：

计算：

```
result = A点数 + B点数
```

结果处理：

* result > 0 → 生成一个红队战士，点数为 result
* result < 0 → 生成一个黑队战士，点数为 result
* result = 0 → 双方战士同时消失

---

### 2. 战士 vs 敌方营地

当战士移动到敌方营地：

计算：

```
result = 战士点数 + 营地点数
```

结果处理：

* result > 0 → 该格变为红队单位（点数 = result）
* result < 0 → 该格变为黑队单位（点数 = result）
* result = 0 → 该格清空

---

### 3. 战士 vs 己方营地（升级）

当战士移动到己方营地：

计算：

```
result = 战士点数 + 营地点数
```

结果：

* 战士消失
* 营地点数更新为 result

---

## 六、胜负判定

当任意一方：

* 所有营地被消灭

则游戏立即结束，对方获胜。

---

## 七、约束规则

* 所有单位必须存在于地图范围内
* 不允许重叠单位（冲突必须立即结算）
* 点数为 0 的单位立即消失
* 每回合只能执行一次行动

---

## 八、数学本质（供策略理解）

游戏本质是：

* 整数加法对抗系统
* 阵营由符号决定
* 资源转化（战士 ↔ 营地强化）
* 随机数驱动的策略博弈
